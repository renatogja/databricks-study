1️⃣ SQL Execution Order + Window Functions

Por que window functions não podem estar em WHERE?
Porque SQL executa em ORDEM ESPECÍFICA: FROM → WHERE → GROUP BY → SELECT → ORDER BY
Window functions calculam no passo SELECT (passo 4), mas WHERE é passo 2!

Solução: Use CTE para calcular ANTES de WHERE.
❌ ERRADO: WHERE ROW_NUMBER() OVER (...) = 1; ✅ CORRETO: WITH ranked AS (SELECT *, ROW_NUMBER() OVER (...) AS rn FROM orders) SELECT * FROM ranked WHERE rn = 1;

2️⃣ CTE vs TEMP TABLE

CTE (WITH): Memória, apenas esta query, sem overhead
TEMP TABLE: Disco, persiste até DROP, mais lento
Para Debugging: CTE é MUITO melhor!
❌ EVITAR: CREATE TEMP TABLE step1 AS SELECT *; ✅ PREFERIR: WITH step1 AS (SELECT * FROM raw_data) SELECT * FROM step1;

3️⃣ ROLLUP vs CUBE vs GROUPING SETS

ROLLUP: Cascata hierárquica (região → loja) ✅
CUBE: TODAS combinações (geralmente inútil) ❌
GROUPING SETS: Você especifica exatamente ✅
UNION ALL: 3x mais lento ❌
GROUP BY ROLLUP(region, store_id) -- Gera: (region, store), (region), () -- 1 scan, hierárquico, perfeito!

4️⃣ Liquid Clustering + OPTIMIZE

DEFINE (CLUSTER BY): Define ESTRATÉGIA nos metadados
APPLY (OPTIMIZE): Reorganiza FISICAMENTE no disco
⚠️ Sem OPTIMIZE, dados continuam desorganizados!
CREATE TABLE orders (...) CLUSTER BY (customer_id); INSERT INTO orders VALUES (...); OPTIMIZE orders; -- ✅ NECESSÁRIO!

5️⃣ Unity Catalog Deny-by-Default

❌ ERRO: "Schemas vazios são invisíveis"
✅ CORRETO: Invisíveis porque FALTA USE SCHEMA privilege!
Um schema pode ter 100 tabelas mas ser INVISÍVEL.
GRANT USE SCHEMA ON SCHEMA prod.analytics TO `user@company.com`; -- Agora aparece em SHOW SCHEMAS!

6️⃣ LAG Function + Window Functions

LAG: Traz valor da linha ANTERIOR
Problema: LAG calcula TODOS gaps (1→2, 2→3, 3→4...)
Solução: Use ROW_NUMBER() + JOIN para posição específica
❌ ERRADO: AVG(DATEDIFF(..., LAG(...))) ✅ CORRETO: WITH ranked AS (SELECT *, ROW_NUMBER() OVER (...) AS rn FROM orders) SELECT AVG(DATEDIFF(r2.order_date, r1.order_date)) FROM ranked r1 JOIN ranked r2 ON r1.customer_id = r2.customer_id AND r1.rn = 1 AND r2.rn = 2;
